## Â§0. The object: a loop of panels

We fold a tessellated plate. Each panel is a unit square; the fold pattern is a **Hamiltonian
circuit** (HC) on the grid graph. Folding it flat should produce compact **stacks**. Two
independent things can go wrong:

1. **Reflection** â€” the cut ends fail to rejoin (stacks unequal). *Separate condition; not
   covered here.*
2. **Twist** â€” even if the ends meet, the loop is **entangled** (self-linked) and cannot lie
   flat. **This is the twist criterion.**

Twist is a property of the *uncut, closed* loop.

## Â§1. Two-stack â€” the foundation

### 1.1 Tessellation â†’ grid graph

Each panel becomes a node; panels sharing a side are joined by an edge. The fold pattern lives
on this graph.

![Tessellation to grid graph](svg/A1_grid_to_graph.svg)

### 1.2 Hamiltonian circuit: creases vs slits

An HC visits every panel exactly once and closes up. The grid edges it **crosses** become
**creases** (folds, red); edges it does **not** cross become **slits** (gray). This turns an
over-constrained origami into a foldable kirigami.

![HC, creases and slits](svg/A2_hc_creases_slits.svg)

### 1.3 Folding = reflection

Folding panel `P_i` flat onto its neighbour `P_{i+1}` across their shared crease is exactly a
**mirror reflection** across the crease line. (Note the corner marker flips side â€” a reflection
reverses orientation.)

![Folding is reflection](svg/A3_fold_is_reflection.svg)

### 1.4 Two reflections = a rotation

Euclidean isometry: composing two reflections across lines meeting at angle `Î±` is a
**rotation by `2Î±`** about their intersection (across *parallel* lines it is a translation).
So folding `P_i â†’ P_{i+1} â†’ P_{i+2}` injects a local in-plane rotation `Î³ = 2Î±`.

![Two reflections make a rotation](svg/A4_two_reflections_rotation.svg)

### 1.5 On a square, the only rotations are Î³ âˆˆ {0, Â±180Â°}

Square creases are axis-aligned, so consecutive creases are either **parallel** (Î± = 0 â†’
Î³ = 0, straight-through) or **perpendicular** (Î± = 90Â° â†’ Î³ = Â±180Â°, an L-turn). The sign
distinguishes a left-L (`+180`) from a right-L (`âˆ’180`). Straight panels contribute nothing;
**only L-corners carry twist**. (Paper Fig 12a.)

![Square gamma cases](svg/A5_gamma_square_cases.svg)

Same rotation magnitude can land `P_i` **under** or **over** `P_{i+2}` depending on fold order

### 1.8 Odd # reflections = a flip = which stack

Each reflection has determinant `âˆ’1`, so after `k` folds the net map has `det = (âˆ’1)^k`.
**Even k** â†’ proper, panel **face-up** (top stack); **odd k** â†’ improper, **face-down**
(bottom stack). *The two stacks are precisely the even/odd reflection-count classes* â€” and Ïƒ
is exactly this parity.

![Odd reflection flip](svg/A8_odd_reflection_flip.svg)

### 1.9 A valid loop is always even-length

The grid graph is **bipartite** (checkerboard), so any closed HC visiting all cells has
**even length `2n`** â†’ an even number of reflections â†’ the loop returns proper (face-up) and
closes consistently. A 2-stack loop can never terminate on a lone odd reflection.

![Even loop bipartite](svg/A9_even_loop_bipartite.svg)

### 1.10 Twist, and a worked foldable case

Each crease sits in two consecutive local rotations (double-counted); summing, halving, and
normalising by `2Ï€`:

$$ Tw = \frac{1}{4\pi} \sum_{i=1}^{2n} g(i)
      = \frac{1}{4\pi}\big(\mathrm{Odd}(\mathcal P) + \mathrm{Even}(\mathcal P)\big) $$

For 2Ã—4 squares the four L-corners give `Odd = âˆ’2Ï€`, `Even = +2Ï€` â†’ `Î£g = 0` â†’ **`Tw = 0`**:
foldable. (Paper Fig 13b.)

![Worked Tw=0](svg/A10_tw_zero_2x4.svg)

### 1.11 A worked twisted case

A square ring (3Ã—3 with the centre removed) gives `Odd = 0`, `Even = +4Ï€` â†’ `Î£g = 4Ï€` â†’
**`Tw = +1`**: entangled, **not** foldable. (Paper Fig 13a.)

![Worked Tw nonzero](svg/A11_tw_nonzero_hole.svg)

### 1.12 Why Tw = 0 is the exact condition â€” CÄƒlugÄƒreanuâ€“Whiteâ€“Fuller

For a closed ribbon, `Lk = Tw + Wr`, where the **linking number `Lk`** is a topological
invariant (cannot change without cutting), and **writhe `Wr`** measures spatial coiling of the
neutral axis.

- Flat start â†’ `Lk = 0`. Rigid folding never cuts â†’ `Lk = 0` throughout.
- Flat stacks â†’ no coiling â†’ `Wr = 0`.
- Therefore `0 = Tw + 0` â†’ **`Tw = 0`**.

Any twist injected forces `Wr = âˆ’Tw â‰  0` (the loop must coil) â†’ entanglement. (Paper Fig 10:
a band twisted by `2Ï€` has `Tw=1, Wr=0`; relaxing trades it to `Tw=0, Wr=1`, but `Lk=1`
throughout â€” the entanglement never leaves without a cut.)

![CWF band](svg/A12_cwf_band.svg)

**Two-stack criterion:** foldable âŸº reflection condition holds **and** `Tw = 0`.

## Â§2. Three-stack 1+1+1 â€” the extension

The 3-stack target splits the footprint into three 1-chains. Everything above is inherited;
only the *loop structure* changes.

### 2.1 Structures: simple loop vs theta graph

A 2-stack fold is a **simple loop** (cycle rank 1). A 1+1+1 fold has two **fused footprint
hubs** joined by three chains â€” a **theta graph** (cycle rank 2). The 2+1 case is the same
theta with one chain a 2-chain (the harder, still-open case).

![Structures compared](svg/B1_structures_compared.svg)

### 2.2 Theta anatomy

The start footprint `S` and exit footprint `E` are **rigid** (their internal creases are
fused). Chains A, B, C each run `S â†’ E`. Cycle rank `= E âˆ’ V + 1 = 3 âˆ’ 2 + 1 = 2`: only **two
independent cycles**.

![Theta anatomy](svg/B2_theta_anatomy.svg)

### 2.3 Pairwise loops

Every cycle in a theta graph is "one chain out, another chain back" â€” a **pairwise loop**
through both hubs. There are three (AB, AC, BC). Each is a genuine closed 2-stack-style loop,
so CWF demands `Tw = 0` on it. The criterion is therefore:

$$ \text{foldable} \iff \mathrm{Tw}(L_{AB}) = \mathrm{Tw}(L_{AC}) = \mathrm{Tw}(L_{BC}) = 0 $$

where each `Tw(L_{ij})` is the paper's closed-loop twist (Â§1.10) of the loop `L_{ij}` = chain
`i` (Sâ†’E) followed by chain `j` (Eâ†’S).

![Pairwise loops](svg/B3_pairwise_loops.svg)

**Test all three â€” twist is not a homology class.** The theta graph has cycle rank 2, so
homologically `[L_{AC}] = [L_{AB}] + [L_{BC}]` and only two loops are independent. But twist is
a *geometric* invariant of an embedded loop, **not** additive over cycle sums:
`Tw(L_{AC}) â‰  Tw(L_{AB}) + Tw(L_{BC})` in general. So the third pairwise twist is genuinely
independent information and **all three must vanish** â€” checking a fixed independent pair would
miss the twisted solutions whose nonzero loop is the omitted one (empirically the twisted pair
is `BC` for the L footprint but `AC` for some Rect cases). This is exactly `search.py twist_check`.

> write `Tw(L_{ij}) = T_i âˆ’ T_j` for a per-chain scalar `T_i`, reducing the criterion to
> `T_A = T_B = T_C`. **That reduction is false** (disproved below in Â§2.5): twist of a pairwise
> loop is not a difference of per-chain potentials.

### 2.4 The chain-end orphan (Q8)

Along an open chain, reflections pair up into `g(i)` terms â€” except the **last** panel at the
hub, a single **unpaired** reflection (improper, no successor crease). This orphan is why a
naÃ¯ve *per-chain* twist false-negatived. **Fused-hub closure** re-pairs the orphan: closing a
chain pair through the rigid hub restores an even, proper closed loop where `Tw` is defined.

![Chain-end orphan](svg/B4_chain_end_orphan.svg)

### 2.5 A tempting reduction that fails â€” global-Ïƒ per-chain twist

It is tempting to collapse Â§2.3 to a *per-chain* number. Define each chain's twist over its
interior corners with a **global** checkerboard sign `Ïƒ = (âˆ’1)^{x+y}`:

$$ T_i = \sum_{\text{interior L-corners of chain }i} \sigma(v)\,\gamma(v), \qquad
   \text{(proposed)}\ \ \text{foldable} \iff T_A = T_B = T_C $$

The idea: each pairwise loop's twist would be `T_i âˆ’ T_j` (the shared Sâ†’E hub rotation
traversed forward in one chain and reverse in the other, cancelling), so foldability reduces to
the three per-chain values agreeing. **This is wrong.** Two independent disproofs, from
`py/analyze_twist.py` over the full cache (68 1+1+1 solutions, 6Ã—6/12Ã—4/9Ã—4/6Ã—4):

1. **Cocycle obstruction (fatal).** If `Tw(L_{ij}) = T_i âˆ’ T_j`, the three pairwise twists must
   satisfy `Tw(L_{AB}) âˆ’ Tw(L_{AC}) + Tw(L_{BC}) = 0`. Twisted solutions have e.g.
   `{AB:0, AC:0, BC:720}` â‡’ the sum is `720 â‰  0`. No per-chain `T_i` can produce that â€” a
   pairwise loop carries twist that is *not* a difference of per-chain potentials. (This is the
   Â§2.3 "twist is not a homology class" point.)
2. **False negatives on foldable solutions.** Foldable pieces (all three pairwise loops `= 0`)
   yield disagreeing per-chain values such as `T = [0, 180, âˆ’180]` or `[180, 0, 0]`. The
   interior-corner sum is not invariant on its own â€” it drops the chain-end **orphan** /
   hub-closure terms (Â§2.4) that carry real twist.

Score: the correct **all-pairwise-loops-zero** criterion matches ground truth **68/68**; the
per-chain `T_A=T_B=T_C` proposal matches only **45/68**. **Use Â§2.3.** The per-chain form is
kept here only as a cautionary record.
